# 결함 회귀 테스트용 샘플

이 문서는 검증기가 코드펜스 내부를 무시하고, 어긋난 ID 헤딩을 HARD 오류로 잡아내는지 확인하기 위한 것이다.

## 형식 예시 (코드펜스 안 — 집계되면 안 됨)

```
### P3-01
- 문제 한줄: 이것은 형식 설명용 예시이며 실제 문제 항목이 아니다
- 당사자: 예시 당사자
- 발생 맥락: 예시 맥락
- 피해 규모: 예시 규모
- 현재 대응: 예시 대응
- 근거 URL: https://example.com/
- 관련 업권: 은행
```

### P3-1
- 문제 한줄: 일련번호가 한 자리라서 형식 오류인 항목이다
- 당사자: 테스트 대상자
- 발생 맥락: 테스트
- 피해 규모: 수치없음
- 현재 대응: 없음
- 근거 URL: https://example.com/
- 관련 업권: 무관

### P3-01: 설명이 붙은 헤딩
- 문제 한줄: 헤딩에 부가 텍스트가 붙어서 형식 오류인 항목이다
- 당사자: 테스트 대상자
- 발생 맥락: 테스트
- 피해 규모: 수치없음
- 현재 대응: 없음
- 근거 URL: https://example.com/
- 관련 업권: 무관

### P3-02
- 문제 한줄: 정상 형식을 모두 갖춘 항목이다
- 당사자: 정상 당사자
- 발생 맥락: 정상 맥락
- 피해 규모: 수치없음
- 현재 대응: 정상 대응
- 근거 URL: https://www.fss.or.kr/
- 관련 업권: 은행

### 참고 사항

이 소제목은 일반 소제목이며 ID 헤딩이 아니므로 HARD 오류로 잡히면 안 된다.
