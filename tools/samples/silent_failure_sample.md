# 조용한 실패 회귀 테스트용 샘플

이 문서는 검증기가 다음 두 가지 조용한 실패 경로를 더 이상 만들지 않는지 확인하기 위한 것이다.

1. 어긋난 ID 헤딩 뒤의 필드가 직전 정상 블록의 필드를 덮어쓰지 않는다.
2. 닫히지 않은 코드펜스가 HARD 오류로 보고되며, 그 안의 완전한 문제 블록도 무시되지 않고 별도로 확인 가능해야 한다.

### P4-01
- 문제 한줄: 정상 항목이며 어긋난 헤딩 뒤 필드에 오염되면 안 된다
- 당사자: 정상 당사자
- 발생 맥락: 정상 맥락
- 피해 규모: 수치없음
- 현재 대응: 정상 대응
- 근거 URL: https://example.com/original
- 관련 업권: 은행

### P4-1
- 문제 한줄: 일련번호가 한 자리라서 형식 오류인 항목이며 이 필드들이 P4-01을 오염시키면 안 된다
- 당사자: 오염 당사자
- 발생 맥락: 오염 맥락
- 피해 규모: 수치없음
- 현재 대응: 오염 대응
- 근거 URL: https://example.com/polluted
- 관련 업권: 증권

여기서부터 닫히지 않은 코드펜스를 연다.

```
### P4-02
- 문제 한줄: 코드펜스 미닫힘으로 인해 유실될 위험이 있는 완전한 형식의 항목이다
- 당사자: 펜스 안 당사자
- 발생 맥락: 펜스 안 맥락
- 피해 규모: 수치없음
- 현재 대응: 펜스 안 대응
- 근거 URL: https://example.com/fenced
- 관련 업권: 보안
